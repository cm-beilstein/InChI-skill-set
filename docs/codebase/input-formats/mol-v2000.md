# MDL MOL V2000 File Format

**Analysis Date:** 2026-04-22

## Overview

The **MDL MOL V2000** format (also known as Molfile V2000 or CTFile V2000) is a text-based chemical file format developed by Molecular Design Limited (MDL, now part of Dassault Systèmes BIOVIA) for representing molecular structures. It stores information about atoms, bonds, connectivity, and 2D/3D coordinates in a connection table format that is readable by most cheminformatics software.

The V2000 format was introduced as part of the MDL CTFile family and became the de facto standard for chemical structure representation. It uses a fixed-field width format where each line contains precisely defined character positions, making it suitable for parsing by automated software. The format is limited to 999 atoms and 999 bonds per molecule, which is sufficient for most small organic molecules but insufficient for large biomolecules like proteins or polymers.

**When to use V2000 vs V3000:** The V2000 format should be used for small to medium-sized molecules (under 1000 atoms) with standard stereochemistry and bonding. The **V3000** (Extended MOLfile) format should be used when:
- The molecule exceeds 999 atoms or 999 bonds
- R-group logic or enhanced stereochemistry is required
- The molecule contains 3D objects or S-groups exceeding V2000 limits

The InChI Software accepts MOL V2000 files as primary input for generating International Chemical Identifiers (InChI).

## File Structure

```
+--------------------------------------------------+
|              HEADER BLOCK (3 lines)                |
|  Line 1: Molecule name                           |
|  Line 2: Comment/user information               |
|  Line 3: Program/tool that created it            |
+--------------------------------------------------+
|              COUNTS LINE (1 line)               |
|  aaabbblllfffcccsssxxxrrrpppnnn               |
+--------------------------------------------------+
|              ATOM BLOCK (n atoms)              |
|  x y z atom_type m d c s h v x x x            |
|  (10 columns, one atom per line)                |
+--------------------------------------------------+
|              BOND BLOCK (m bonds)              |
|  a1 a2 type stereo topo react                  |
|  (6 columns, one bond per line)                |
+--------------------------------------------------+
|           PROPERTIES BLOCK (optional)            |
|  M CHG, M RAD, M ISO, M ALS, etc.             |
+--------------------------------------------------+
|                 M END                         |
+--------------------------------------------------+
```

## Header Block Detail

The header block consists of exactly 3 lines that provide metadata about the molecule and the file source.

### Line 1: Molecule Name

**Position:** Columns 1-44 (max 44 characters)
**Contents:** The name of the molecule, which may include alphanumeric characters, spaces, and common punctuation.
**Example:**
```
Benzene
```

This line can be left blank (a blank line must still be present), which is common when the molecule name is defined elsewhere, such as in an SDF file's name field.

### Line 2: Comment/User Information

**Position:** Columns 1-44 (max 44 characters)
**Contents:** User-defined comments or additional information about the molecule. This line is often used for:
-timestamps (e.g., "Mon Jul 17 14:30:00 2023")
-software-specific metadata
-文献 reference
- Any user annotation

**Example:**
```
Created by MarvinScan 5.9.4, 17/07/23
```

This line can also be blank, but must exist in the file.

### Line 3: Program/Tool That Created File

**Position:** Columns 1-44 (max 44 characters)
**Contents:** Contains the program name and version that generated the file, plus an optional timestamp in the format `MMDDYY` or `YYMMDD`.
**Example:**
```
  ACSII  1.0.0                 11020214353D
```

Common program codes include:
- `CHEMVIEW` - ChemView
- `MARVIN` - ChemAxon Marvin
- `ISIS` - MDL ISIS/Draw
- `OEB` - OpenEye Babel
- `ACDLAB` - ACD/Labs
- `BIOVIA` - BIOVIA products

The timestamp format varies by tool but typically encodes date information in the final characters.

## Counts Line Detail

The counts line (also called the counts header or CTAB line) immediately follows the header block and defines the number of atoms, bonds, and other structural elements.

**Format:** `aaabbblllfffcccsssxxxrrrpppnnn`

| Position | Field | Width | Description |
|----------|-------|-------|-------------|
| 1-3 | `aaa` | 3 | **Number of atoms** (001-999). The number of atom lines that follow in the atom block. |
| 4-6 | `bbb` | 3 | **Number of bonds** (000-999). The number of bond lines that follow in the bond block. |
| 7-9 | `lll` | 3 | **Number of atom lists** (000-999). Defines atom lists for query features. |
| 10-12 | `fff` | 3 | **Number of 3D objects** (000-999). For additional structural features. |
| 13-15 | `ccc` | 3 | **Chiral flag** (0 or 1). 0 = not chiral, 1 = chiral. Indicates all stereocenters have absolute configurations. |
| 16-18 | `sss` | 3 | **Number of stereotext entries** (000-999). Text definitions for stereochemistry. |
| 19-21 | `xxx` | 3 | **Number of lines of additional properties** (000-999). Lines in the properties block (after M END would appear). |
| 22-24 | `rrr` | 3 | **Flag for registration number** (000-999). Reserved for database registration. |
| 25-27 | `ppp` | 3 | **Number of links** (000-999). For reactions and linked structures. |
| 28-30 | `nnn` | 3 | **Number of explicit hydrogens** counted on atoms with the "H" property set. |
| 31-34 | - | 4 | **Version string** (usually "V2000"). Must read "V2000" for proper parsing. |

**Example Counts Line:**
```
  6  5  0  0  1  0  0  0  0  0999 v2000
```

Breakdown:
- `6` = 6 atoms (carbon atoms in benzene)
- `5` = 5 bonds (the ring bonds between carbons)
- `0` = no atom lists
- `0` = no 3D objects
- `1` = chiral flag set (benzene has defined stereochemistry)
- `0` = no stereotext entries
- `0` = no additional property lines
- `0` = no registration number
- `0` = no links
- `999` = placeholder/max value
- `v2000` = version indicator

## Atom Block Detail

Each line in the atom block represents one atom with 10 fixed-position columns. The format is:

```
x y z atom_type m d c s h v 0 0 0
```

| Column | Width | Field | Description |
|--------|-------|-------|-------------|
| 1-10 | 10 | `x` | **X-coordinate** (floating point, position in Ångströms or 2D units) |
| 11-20 | 10 | `y` | **Y-coordinate** (floating point) |
| 21-30 | 10 | `z` | **Z-coordinate** (floating point, often 0.0000 for 2D structures) |
| 31-34 | 4 | `atom_type` | **Atom symbol** (1-3 characters): Standard elements (C, N, O, S),卤素 (Cl, Br, I), metals (Fe, Na, K), or special atoms |
| 35-36 | 2 | `m` | **Mass difference** (-5 to +5, or 0 for natural abundance): Difference from nominal mass number. Negative = lighter isotope, positive = heavier |
| 37-38 | 2 | `c` | **Charge** (0 = none/unknown, 1 = +1, 2 = +2, 3 = +3, 4 = -1, 5 = -2, 6 = -3, 7 = doublet radical) |
| 39-40 | 2 | `s` | **Stereo parity** (0 = unspecified, 1 = odd, 2 = even, 3 = either, or "0" to ignore). Note: Ignored by most software. |
| 41-42 | 2 | `h` | **H count + 1**: Explicit hydrogens attached (displayed as 0-3 meaning actual 1-4). Value is hydrogen count + 1. |
| 43-44 | 2 | `v` | **Valence** (0 = valence is determined automatically, 1-14 for specific valences) |
| 45-46 | 2 | (reserved) | Reserved, always 0 |
| 47-48 | 2 | (reserved) | Reserved, always 0 |
| 49-50 | 2 | (reserved) | Reserved, always 0 |

### Atom Type Symbols

**Standard Elements:**
- Periodic table elements: H, He, Li, Be, B, C, N, O, F, Ne, Na, Mg, Al, Si, P, S, Cl, Ar, K, Ca, Sc, Ti, V, Cr, Mn, Fe, Co, Ni, Cu, Zn, Ga, Ge, As, Se, Br, Kr, etc.
- Atomic numbers are also accepted: 1H, 6C, 7N, 8O, etc.

**Query/Generic Atoms:**
- `A` - Any atom except hydrogen
- `Q` - Any atom except carbon or hydrogen
- `*` - Any atom (matches X in SMILES)
- `LP` - Lonely pair (lone electron pair)
- `L` - Atom list (see M ALS property)
- `L[abc]` - Exclusive atom list (atoms NOT in list)
- `R#` or `R` - R-group attachment point (R is numbered R1, R2, etc. when followed by number)

### Charge Codes

| Code | Meaning |
|------|---------|
| 0 | No charge or unknown |
| 1 | +3 charge |
| 2 | +2 charge |
| 3 | +1 charge |
| 4 | -1 charge |
| 5 | -2 charge |
| 6 | -3 charge |
| 7 | Doublet radical (e.g., CH3•) |

Some software stores exact charges in the M CHG property block instead.

## Bond Block Detail

Each line in the bond block represents a chemical bond between two atoms. The format is:

```
a1 a2 type stereo topo react
```

| Column | Width | Field | Description |
|--------|-------|-------|-------------|
| 1-3 | 3 | `a1` | **First atom** number (1 to n, where n is from counts line) |
| 4-6 | 3 | `a2` | **Second atom** number |
| 7-9 | 3 | `type` | **Bond type** (1=single, 2=double, 3=triple, 4=aromatic, etc.) |
| 10-12 | 3 | `stereo` | **Bond stereo** (0=none, 1=up, 2=down, 3=cumulen) |
| 13-15 | 3 | `topo` | **Bond topology** (0=either, 1=ring, 2=chain) |
| 16-18 | 3 | `react` | **Reacting center status** (0=either, -1=not center, 1=center) |

### Bond Type Codes

| Code | Type | Description |
|------|------|-------------|
| 1 | Single bond | sigma bond |
| 2 | Double bond | one sigma + one pi bond |
| 3 | Triple bond | one sigma + two pi bonds |
| 4 | Aromatic bond | aromatic system bond (benzene ring) |
| 5 | Single or double | Query bond (either single or double) |
| 6 | Single or aromatic | Query bond |
| 7 | Double or aromatic | Query bond |
| 8 | Hydrogen bond | Special H-bond indicator |
| 9 | Distance (3-pt) | For stereochemistry |
| 10 | Distance (4-pt) | For stereochemistry |
| 11 | No bond | Null bond (for queries) |

### Bond Stereo Codes

| Code | Meaning |
|------|---------|
| 0 | No stereochemistry specified |
| 1 | Up (bond points toward viewer) |
| 6 | Down (bond points away from viewer) |
| 3 | Either (cis/trans mixture) |

The stereo field is primarily used in conjunction with stereocenter atoms and the chiral flag to define cis/trans (E/Z) isomers.

### Bond Topology Codes

| Code | Meaning |
|------|---------|
| 0 | Either (can be ring or chain) |
| 1 | Ring bond (part of a ring) |
| 2 | Chain bond (not in a ring) |

### Reacting Center Status Codes

| Code | Meaning |
|------|---------|
| -1 | Not a reacting center |
| 0 | Unspecified/unknown |
| 1 | Reacting center |

This field is used primarily in reaction files to indicate which atoms participate in reactions.

### Example Bond Lines (Benzene)

```
  1  1  4  0  1  0
  1  2  4  0  1  0
  1  3  4  0  1  0
  1  4  4  0  1  0
  1  5  4  0  1  0
```

These represent the aromatic bonds in benzene's six-membered ring, where each bond is type 4 (aromatic), with topology 1 (ring bond), and no reacting center status.

## Properties Block Detail

The properties block contains additional molecular properties that cannot be stored in the atom or bond blocks. Each property line begins with `M ` followed by a two-letter property code.

### M CHG - Atomic Charge

**Format:** `M CHG n a1 c1 a2 c2 ...`
- `n` = number of charge records that follow
- Each pair: `atom_number` and `charge_value`

**Example:**
```
M CHG 1  1  3
```
Means: Atom 1 has +3 charge (or code 3, meaning +1 in some interpretations).

### M RAD - Atomic Radical

**Format:** `M RAD n a1 r1 a2 r2 ...`
- `n` = number of radical records
- Each pair: `atom_number` and `radical_type`

**Radical Types:**
- 1 = Singlet (two electrons)
- 2 = Doublet (one electron)
- 3 = Triplet (two electrons)

**Example:**
```
M RAD 1  1  2
```
Means: Atom 1 has a doublet radical.

### M ISO - Isotope Specification

**Format:** `M ISO n a1 i1 a2 i2 ...`
- `n` = number of isotope records
- Each pair: `atom_number` and `isotope_mass`

**Example:**
```
M ISO 1  1  13
```
Means: Atom 1 is Carbon-13 (mass number 13).

### M ALS - Atom List (Query Feature)

**Format:** `M ALS n a s t f1 f2 f3 ...`
- `n` = atom list number
- `a` = target atom number
- `s` = list type (0 = atom list, 1 = exclusive list)
- `t` = number of atom types in list
- `f1, f2, f3...` = atom symbols or atomic numbers

**Example:**
```
M ALS 1  5 0 3 C N O
```
Means: Atom 5 is bonded to any of (C, N, or O).

### M STY - Sgroup Type (Superatom)

**Format:** `M STY n s1 t1 s2 t2 ...`
- `n` = number of S-groups
- Each record: S-group number and type (SRU, POL, MER, COL, etc.)

### M SBL - Sgroup Bond

**Format:** `M SBL n b1 a1 b2 a2 ...`
- Connects atoms to S-groups

### M SLB - Sgroup Link

**Format:** `M SLB n s1 l1 s2 l2 ...`
- Links between S-groups

### M SCN - Sgroup Component Number

**Format:** `M SCN n c1 c2 ...`
- Component numbers for data S-groups

### Common Query Properties

| Property | Use |
|----------|-----|
| M SUB n a1 s1 a2 s2... | Substitution count |
| M UNS n a1 u1 a2 u2... | Unsaturated atom |
| M RBC n a1 r1 a2 r2... | Ring bond count |
| M ALS | Atom list query |
| M APO | R-group attachment |
| M RGP | R-group labels |
| M LOG | R-group logic |
| M LIN | Link nodes |

## Implementation Notes for InChI

### Reading MOL V2000 into InChI

1. The InChI software parses the counts line to determine memory allocation requirements
2. Atom block coordinates are used for 2D/3D structure perception and canonicalization
3. Bond block determines connectivity and bond order for the connectivity layer
4. Properties block (M CHG, M ISO, M RAD) affect the charge and isotope layers
5. Aromatic bond types (4) are normalized to alternating single/double during conversion

### Limitations for InChI Generation

1. **Maximum atoms:** 999 (V2000 limit) - molecules larger require V3000 format with InChI's large molecule option
2. **Isotopes:** Natural isotopes inferred from mass difference field; metastable isotopes (e.g., Tc-99m) not supported
3. **Stereochemistry:** InChI interprets the chiral flag and bond stereo fields to generate stereochemical layers
4. **Implicit hydrogens:** Some software outputs explicit hydrogens; InChI accounts for implicit hydrogens based on valence rules
